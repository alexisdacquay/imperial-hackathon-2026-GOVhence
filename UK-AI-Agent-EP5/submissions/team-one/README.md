## The problem

```
   Many AI agents.            One shared memory.          A leak waiting to happen.
         o  o  o        в”Җв”Җв”Җв–ә        (everything)     в”Җв”Җв”Җв–ә       driver's bot reads
         o  o  o                                                the CEO's financials
```

Shared memory makes agents smarter вҖ” they stop repeating work and share context. But pool
everything and the logistics driver's bot can read the CEO's financials. The track asks for
memory that is **shared but fenced** вҖ” same brain, per-role pages вҖ” with **deterministic
enforcement**, **lineage governance for derived memory**, and an **audit trail**.

## Our approach

LLMs can be tricked, jailbroken, or just wrong вҖ” so **no LLM ever touches a permission
decision**. LLMs do the *language* work (what is this message about? is it worth storing?);
**plain auditable code** does all the *power* work:

- Every memory **CARRIES `labels`** (security labels: `financials`, `legal`, вҖҰ). Every user
  **HOLDS `clearances`** via their role ([data/users.json](data/users.json)).
- **The gate is one line of set maths:** served вҮ” `labels вҠҶ clearances` вҖ” the reader must hold
  **ALL** of a memory's labels. Whitelist-only: no deny lists, no wildcards; a memory with no
  labels is visible to **nobody** (fail-closed, not fail-open).
- `topics` (what a memory is *about*, produced by an LLM) are used **only for relevance** вҖ”
  a topic can never grant access, so nothing can be smuggled past the gate via classification.

## Orchestration flow

Hub-and-spoke: **GOVhence**, a deterministic script, is the sole orchestrator вҖ” every hop goes
through it, so every hop is recorded. Amber = plain code (holds all the power). Green = LLMs
(never decide access). The bundled **Audit Console** ([GUI/](GUI/)) renders this exact flow live.

## What it does вҖ” the feature lineup

*Everything below is implemented, in [src/](src/), and covered by the 109-test suite.*

### 1 В· The Bouncer вҖ” no AI in the access decision
One strict subset check decides every item: `labels вҠҶ clearances`, exact strings, ALL labels
required. Same input, same answer, every time вҖ” and it cannot be talked out of a "no".

### 2 В· Fail-closed, everywhere
Unknown user вҶ’ nothing. Memory with no labels вҶ’ visible to **nobody**. Malformed config,
memory, lineage or identity вҶ’ loud `ConfigError`, nothing served. A broken event log never
blocks a decision; a broken *permission* source always does.

### 3 В· Topics can't smuggle access
Two vocabularies, never mixed: LLM-produced `topics` = relevance only; `labels`/`clearances` =
access only. Querying with `financials` as a *topic* grants exactly nothing вҖ” tested.

### 4 В· Lineage вҖ” derived memory is governed by its sources
```
  inherit:    labels(derivative)  =  labels(own) вҲӘ labels(every source)

              m-revenue {financials}        m-contract {legal}
                         в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                                    в–ј  derived_from
                       m-briefing {financials, legal}     вҶҗ carries BOTH constraints

  who sees the briefing?
              exec      holds {вҖҰ, financials}          вң— DENY   missing legal
              counsel   holds {вҖҰ, legal}               вң— DENY   missing financials
              auditor   holds {вҖҰ, financials, legal}   вң“ ALLOW  holds every label

  revoke:     lineage.json {"revoked": ["m-revenue"]}
              m-revenue в”Җв”Җв–ә m-briefing в”Җв”Җв–ә (anything derived from it в”Җв”Җв–ә вҖҰ)
                DENY            DENY                    DENY
              transitive В· cycle-safe В· "revoked beats permitted" вҖ”
              even the auditor loses it, and the store itself is never touched
```
A summary inherits the access constraints of **all** its sources (label union = you must satisfy
every source's restriction), and revoking a source propagates to every derivative
([src/bouncer.py](src/bouncer.py) `revoked_ids`).

### 5 В· Governed writes вҖ” poisoning defence
The **Judge** decides *if* something is worth storing; the **Memoriser** labels it at write
time from the known vocabulary вҖ” then deterministic code enforces the **writer cap**: you can
never plant content **above your own clearance** (a driver cannot inject into the financials
lane). Invalid labels, LLM outage, or a disk failure вҶ’ the write is **refused**, never stored
mislabelled. Persistence is atomic (write-tmp, then rename).

### 6 В· Live ACL sync вҖ” no restarts
Clearances are cached in memory with a short TTL: edit [data/users.json](data/users.json) and a
revoked clearance stops working within ~5 seconds, on a **running** system. Credential reads
cost ~0.0003 ms warm вҖ” and every one is logged with its duration.

### 7 В· Every decision on the record вҖ” and a GUI to watch it
Each run appends to an event feed ([data/events.jsonl](data/)): every hop with milliseconds,
every credential check, every per-memory **ALLOW/DENY with its reason**. Denied bodies are
**redacted** (`[withheld]`) вҖ” the feed can never become the leak. The **Audit Console**
(`python GUI/serve_gui.py`) graphs the pipeline and streams the Bouncer's decisions live.

### 8 В· No silent fallbacks
If the Judge's model is down, GOVhence **refuses the message loudly** вҖ” it never quietly
degrades to keyword rules that could mis-govern. Transient empty LLM replies are retried;
persistent failure is an honest error, not a guess.
