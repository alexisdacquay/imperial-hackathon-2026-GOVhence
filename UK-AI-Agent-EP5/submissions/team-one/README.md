## The problem, in one breath

```
   Many AI agents.            One shared memory.          A leak waiting to happen.
         o  o  o        в”Җв”Җв”Җв–ә        (everything)     в”Җв”Җв”Җв–ә       driver reads the
         o  o  o                                                 CEO's financials
```

Shared memory makes agents smarter вҖ” they stop repeating work and share context. But put
**everything in one place** and suddenly the logistics driver's bot can read the CEO's financials,
and a contractor's agent can open confidential legal files.

**The mission:** keep the memory **shared but fenced** вҖ” everyone reads the same brain, but each
person only ever sees the pages their role unlocks. And you must be able to **prove it** to a
regulator.

## The big idea

```
   в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
   в”Ӯ   The AI NEVER decides who gets access.   Plain code does.    в”Ӯ
   в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

- An AI can be **tricked, jailbroken, or hallucinate** вҶ’ never trust it with the keys.
- A **deterministic code check** can't be talked out of a "no." Same input, same answer, every time.
- Every memory item wears a **category tag** (`schedules`, `financials`, `legal`вҖҰ). Your role
  unlocks a set of tags. Access = *is this tag in your set?* вҖ” an **exact, strict** match.
- Every decision is **logged immutably**. Revoke a source and the block **chases it everywhere**.
- **We verify by trying, not by trusting.** A real guard doesn't glance at the chained gate and
  assume it's locked вҖ” they *rattle it*. Our security actively attacks itself: it probes its own
  defences for ways in, rather than asserting they hold. Active verification over passive trust.

> This is exactly what the track demands: *"enforcement must be deterministic."* We took it literally.

## What it does вҖ” the feature lineup

> Every feature below is **live and covered by automated tests** (the suite passes, exit 0). вң…

### 1 В· Deterministic access control вҖ” *the bouncer*
One strict, exact comparison decides every item. **No AI in the decision.** Identical every time,
fully auditable.

### 2 В· Fail-closed by *default*
Unknown tag? Malformed input? Bad config? Look-alike Unicode trickery? **вҶ’ DENY.**
- Never "fails open" (never accidentally grants)
- Battle-tested against adversarial inputs вҖ” wrong data types, homoglyphs, empty profiles
- When in doubt, **withhold** вҖ” leaking is never an option

### 3 В· Role-based permissions, *outside the code*
Who-sees-what lives in a plain editable file ([users.json](users.json)).
- Roles вҶ’ allowed categories
- "Allow everything **except** X" (e.g. an exec who sees all *but* legal)
- Admins change access by editing one file вҖ” **zero code changes**

### 4 В· Config that *catches its own mistakes*
- Unknown user / unknown role / typo'd category / corrupt file вҶ’ **stops with a clear error**
- A misconfig in a governance tool gets **flagged**, never silently waved through

### 5 В· Lineage revocation вҖ” *the showstopper*
```
   revoke  source                                    the block PROPAGATES
            в”Ӯ  в””в”Җderivedв”Җв–ә summary                    to every derivative,
            в”Ӯ              в””в”Җderivedв”Җв–ә embedding       however many hops deep
            в–ј
        all of them denied вҖ” even if their tag was "allowed"
```
- Revoke a **source** вҶ’ every item **derived from it** is revoked too, transitively
- **"Revoked beats allowed"** вҖ” a revoked item is denied even if its category is permitted
- This is the requirement **most systems get wrong**. We don't.

### 6 В· Regulator-grade audit log
- **Every** decision logged вҖ” grants **and** refusals
- Captures **who В· what В· when (UTC) В· outcome В· why**
- Append-only В· **no sampling, ever** вҖ” the *complete* record is retained
- Reconstruct *"who saw what, when"* for any item вҖ” including blocked attempts

### 7 В· Tamper-evident вҖ” *the log can't be faked*
```
   row 0 в”Җв”Җhashв”Җв”Җв–ә row 1 в”Җв”Җhashв”Җв”Җв–ә row 2 в”Җв”Җhashв”Җв”Җв–ә row 3
                         в–І
            edit / delete / reorder ANY row  в”Җв”Җв–ә  chain breaks  в”Җв”Җв–ә  detected
```
- Sequence numbers вҶ’ a deleted record leaves a **visible gap**
- SHA-256 **hash chain** вҶ’ any edit/insert/reorder is **caught**, pointing to the exact row
- Built-in `verify()` integrity check вҖ” **court-credible** evidence
- Standards-aligned (NIST integrity / WORM-alternative) вҖ” **no database, no blockchain needed**
- **Checkpoint / running-tally** вҶ’ catches **end-truncation** (rows shaved off the tail) a bare chain can't, keeps appends **O(1)** at any size, and mirrors the latest tally to an off-host **anchor** (the Certificate-Transparency / CloudTrail / QLDB pattern)

### 8 В· No log вҶ’ no access
- If a decision can't be recorded, the access is **refused**
- On a 100%-coverage system, an *unlogged* access is a breach вҖ” so we forbid it

### 9 В· Injection-safe audit trail
- Malicious values (e.g. starting with `=`) can't run as spreadsheet formulas
- Can't forge or corrupt records вҖ” **reviewing the log is safe**
